# VP-007: Digest Stability — Structurally-Equal Policies Produce the Same Digest (DI-018)

## Property Statement

`digest::policy_digest(policy: &Policy) -> String` is a pure function that produces
the same lowercase 64-character SHA-256 hex string for any two `Policy` values that
are structurally equal (same zones and conduits, modulo key order, whitespace, and
duplicate conduits). Specifically:

1. Permuting the `zones` list does not change the digest.
2. Permuting the `conduits` list does not change the digest.
3. Duplicating a conduit does not change the digest (dedup is applied).
4. Two different YAML representations of the same semantic policy produce the same digest.
5. Any change to a zone id, name, purdue_level, sl_t, member, or conduit field
   produces a different digest (with overwhelming probability — collision resistance
   inherited from SHA-256).

## Source Contract

- **BC:** BC-1.05.003 — Stable policy digest via canonical JSON serialization (SHA-256)
- **Postcondition/Invariant:** DI-018 (stable policy digest; canonicalization defined)

## Proof Method

| Method | Tool | Bounded? | Coverage |
|--------|------|----------|----------|
| proptest | proptest | No bound needed | Random permutations; duplicate-conduit insertion; field mutations |

**Note:** SHA-256 internal state has 256-bit width — not feasible for Kani. proptest
is the correct tool: it generates many random `Policy` values, permutes them, and
asserts digest equality. It also mutates single fields and asserts digest change
(with overwhelming probability).

## Proof Harness Skeleton

```rust
use proptest::prelude::*;

proptest! {
    #[test]
    fn vp007_digest_stable_under_zone_permutation(
        policy in arb_policy()
    ) {
        let d1 = digest::policy_digest(&policy);

        // Permute zones
        let mut policy2 = policy.clone();
        policy2.zones.reverse();
        let d2 = digest::policy_digest(&policy2);

        prop_assert_eq!(d1, d2, "digest changed under zone permutation");
    }

    #[test]
    fn vp007_digest_stable_under_conduit_permutation(
        policy in arb_policy()
    ) {
        let d1 = digest::policy_digest(&policy);

        let mut policy2 = policy.clone();
        policy2.conduits.reverse();
        let d2 = digest::policy_digest(&policy2);

        prop_assert_eq!(d1, d2, "digest changed under conduit permutation");
    }

    #[test]
    fn vp007_digest_stable_under_conduit_dedup(
        policy in arb_policy()
    ) {
        let d1 = digest::policy_digest(&policy);

        // Add a duplicate conduit
        let mut policy2 = policy.clone();
        if let Some(c) = policy2.conduits.first().cloned() {
            policy2.conduits.push(c);
        }
        let d2 = digest::policy_digest(&policy2);

        prop_assert_eq!(d1, d2, "digest changed when duplicate conduit added");
    }
}
```
