# VP-004: Verdict Totality — Every Flow Gets Exactly One VerdictKind (DI-015)

## Property Statement

`classifier::classify(ctx, flow, resolved_pair) -> Verdict` always returns exactly one
`VerdictKind` value and never panics. The function is total over all valid `ResolvedPair`
and `ClassifyCtx` inputs.

Specifically:
- `MulticastExempt` is returned iff `resolved_pair.dst.match_kind == MulticastBroadcast` (highest precedence).
- `IntraZone` is returned iff both endpoints resolve to the same zone (and dst is not multicast).
- `Allowed(conduit_idx)` is returned iff at least one conduit permits the flow.
- `NoMatchingConduit` or `WrongDirection` is returned for all other cases.
- No fourth option; no `unreachable!()` panic.

## Source Contract

- **BC:** BC-1.04.010 — Verdict totality: every resolved flow receives exactly one VerdictKind
- **Postcondition/Invariant:** DI-015 (verdict totality and accounting)
- **Also covers:** BC-1.04.001 (intra-zone), BC-1.04.002 (deny-by-default), BC-1.04.011 (multicast precedence)

## Proof Harness Skeleton

```rust
#[cfg(kani)]
#[kani::proof]
#[kani::unwind(64)]
fn vp004_verdict_totality() {
    // Generate symbolic valid policy context (bounded conduits)
    let ctx: ClassifyCtx = ClassifyCtx::kani_any_bounded(16);
    kani::assume(ctx.is_valid());

    // Generate any resolved pair
    let resolved: ResolvedPair = kani::any();
    let flow: Flow = kani::any();

    // Execute
    let verdict = classifier::classify(&ctx, &flow, &resolved);

    // Postcondition 1: exactly one VerdictKind (enum is exhaustive by Rust type system)
    // The proof asserts no unreachable arm is hit — Kani checks for panics automatically

    // Postcondition 2: MulticastBroadcast → always MulticastExempt
    if resolved.dst.match_kind == MatchKind::MulticastBroadcast {
        assert!(matches!(verdict.kind, VerdictKind::MulticastExempt));
    }

    // Postcondition 3: IntraZone iff same zone (and not multicast)
    if resolved.src.zone_id == resolved.dst.zone_id
        && resolved.dst.match_kind != MatchKind::MulticastBroadcast
    {
        assert!(matches!(verdict.kind, VerdictKind::IntraZone));
    }

    // Postcondition 4: Deny-by-default — no matching conduit → NoMatchingConduit or WrongDirection
    // (not Allowed and not IntraZone and not MulticastExempt)
    // This is covered by the type exhaustiveness + the above two assertions
}
```
