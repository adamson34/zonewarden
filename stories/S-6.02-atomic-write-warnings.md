# S-6.02: Reporter — Atomic Write + Deterministic Warnings

## Narrative
- **As a** zonewarden user
- **I want to** have output files written atomically (no partial output on error) and warnings emitted to stderr in deterministic order without affecting the exit code
- **So that** CI pipelines cannot observe corrupt partial output files and warnings are reproducible and predictable

## Acceptance Criteria

### AC-001 (traces to BC-1.06.008 postcondition — atomic write)
When `--output <PATH>` is provided, `reporter::emit_to_file` writes to a temporary file in the same directory as `<PATH>`, then renames atomically to `<PATH>`. On error during write, the target file is not created or left partially written.
- **Test:** `test_BC_1_06_008_atomic_write_uses_temp_then_rename()`

### AC-002 (traces to BC-1.06.008 postcondition — no partial file on error)
If writing fails mid-stream (simulated via a mock that fails after N bytes), the target `<PATH>` does not exist afterward. The temporary file is cleaned up.
- **Test:** `test_BC_1_06_008_no_partial_file_on_write_error()`

### AC-003 (traces to BC-1.06.005 postcondition — warnings to stderr)
Warnings (from `ConformanceResult::warnings`) are written to stderr, not stdout. The main report goes to stdout (or `--output` file).
- **Test:** `test_BC_1_06_005_warnings_to_stderr()`

### AC-004 (traces to BC-1.06.005 postcondition — deterministic order)
Warnings are emitted in the order they appear in `ConformanceResult::warnings`. The warnings vec is ordered by the aggregator (insertion order during ingest, which is deterministic for a fixed input file).
- **Test:** `test_BC_1_06_005_warnings_emitted_in_order()`

### AC-005 (traces to BC-1.06.005 invariant — warnings do not affect exit code)
Emitting warnings (skipped flow warnings, zero-member zone warnings) does NOT change the exit code. Exit code is determined solely by `distinct_violating_flows`, `idmz_bypasses`, and error conditions.
- **Test:** `test_BC_1_06_005_warnings_do_not_change_exit_code()`

### AC-006 (traces to BC-1.06.007 postcondition — offline invariant)
`reporter` does not open any network socket. Verified structurally: no `std::net` import, no `TcpStream`, no `UdpSocket` in `reporter.rs` or any file it imports.
- **Test:** `grep -r "std::net" zonewarden/src/reporter.rs` returns no output (CI lint check)

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | `--output` points to read-only directory | `Err(E-IO-005)` — temp file creation fails; exit 2 |
| EC-002 | `--output` not specified | Write to stdout directly (no temp file) |
| EC-003 | Zero warnings | No stderr output |
| EC-004 | Disk full during rename | Error propagated as `E-IO-006`; target not corrupted |
| EC-005 | Warning with special characters (`\n`, `"`) | Emitted as-is; no escaping |
