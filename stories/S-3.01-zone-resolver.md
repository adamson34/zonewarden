# S-3.01: Zone Resolver — Sorted-Prefix Index + Longest-Prefix Match

## Narrative
- **As a** zonewarden classifier
- **I want to** resolve every flow endpoint IP address to exactly one zone using longest-prefix match on the validated policy index
- **So that** the classifier operates on zone pairs rather than raw IPs, with total and deterministic resolution guaranteed

## Acceptance Criteria

### AC-001 (traces to BC-1.03.001 postcondition 1)
`resolver::resolve(index, ip)` returns a `ResolvedEndpoint` with `zone_id` set to the zone owning the longest-matching prefix. For endpoint `10.0.1.5` with Zone A `10.0.0.0/16` and Zone B `10.0.1.0/24`, returns Zone B.
- **Test:** `test_BC_1_03_001_longest_prefix_wins()`

### AC-002 (traces to BC-1.03.001 postcondition 2)
Resolution is deterministic: identical `(ip, index)` inputs always produce the same `ResolvedEndpoint`.
- **Test:** `test_BC_1_03_001_resolution_deterministic()` (proptest)

### AC-003 (traces to BC-1.03.001 edge case EC-002)
A `/32` host address in Zone A wins over a `/24` network in Zone B when both match.
- **Test:** `test_BC_1_03_001_host_32_wins_over_net_24()`

### AC-004 (traces to BC-1.03.002 postcondition)
An endpoint IP that matches no declared zone matcher resolves to the implicit `EXTERNAL` zone with `match_kind: MatchKind::Implicit`.
- **Test:** `test_BC_1_03_002_unmatched_resolves_to_external()`

### AC-005 (traces to BC-1.03.005 postcondition)
When both flow endpoints resolve to `EXTERNAL`, the resulting flow verdict is `IntraZone` (not `NoMatchingConduit`). This is the both-EXTERNAL special case per DI-002/DI-005.
- **Test:** `test_BC_1_03_005_both_external_yields_intra_zone()`

### AC-006 (traces to BC-1.03.001 invariant 1 — totality)
Resolution is total: for any non-unspecified, non-multicast endpoint IP, `resolver::resolve` always returns a `ResolvedEndpoint` (never panics or returns an error). This is the DI-003 guarantee.
- **Test:** `test_BC_1_03_001_resolution_total_kani_stub()` (Kani harness)

### AC-007 (traces to BC-1.03.001 invariant 3 — pure function)
`resolver::resolve` is a pure function: it takes `(&PrefixIndex, IpAddr)` by reference and returns a value. No mutable state, no I/O.
- **Test:** Compile-time: function signature; `#[deny(clippy::unwrap_used)]` passes

### AC-008 (traces to BC-1.03.001 edge case EC-007 — OQ-005)
Endpoint `10.0.0.0` (network address of zone `10.0.0.0/24`) resolves to that zone normally. No network-address exemption.
- **Test:** `test_BC_1_03_001_network_address_resolves_normally()`

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | Zone A `10.0.0.0/16`, Zone B `10.0.1.0/24`; endpoint `10.0.1.5` | Resolves to Zone B (longer prefix) |
| EC-002 | Zone A `10.0.0.5/32`, Zone B `10.0.0.0/24`; endpoint `10.0.0.5` | Resolves to Zone A (/32 wins) |
| EC-003 | No zone matches `192.168.1.5` | Resolves to EXTERNAL (MatchKind::Implicit) |
| EC-004 | `127.0.0.1` (loopback); no zone claims it | Resolves to EXTERNAL |
| EC-005 | Zone A: `0.0.0.0/8`; endpoint `0.0.0.1` | Resolves to Zone A (matches /8; only /0 is rejected at validation) |
| EC-006 | IPv6 endpoint; policy IPv4-only | EXTERNAL |
| EC-007 | Both src and dst resolve to EXTERNAL | Both EXTERNAL → IntraZone short-circuit (AC-005) |
| EC-008 | Network address `10.0.0.0` with zone `10.0.0.0/24` | Resolves to zone normally |
