# S-5.02: Aggregator ‚Äî ConformanceResult, DI-015 Identity, Overflow Guard

## Narrative
- **As a** zonewarden pipeline
- **I want to** accumulate per-flow verdicts into a `ConformanceResult` with checked arithmetic and a provably-correct accounting identity
- **So that** every run produces a mathematically-verified tally where no flow is lost or double-counted, and u64 overflow is caught before producing an incorrect result

## Acceptance Criteria

### AC-001 (traces to BC-1.05.001 postcondition 1 ‚Äî DI-015 accounting identity)
`aggregator::aggregate` returns a `ConformanceResult` where `total_flows == intra_zone + allowed + no_matching_conduit + wrong_direction + multicast_exempt` (exact u64 equality). Verified for any mix of verdict kinds.
- **Test:** `test_BC_1_05_001_di015_identity_holds()` (proptest)

### AC-002 (traces to BC-1.05.001 postcondition 2 ‚Äî idmz_bypasses ‚â§ total_flows)
`idmz_bypasses ‚â§ total_flows` holds for every ConformanceResult.
- **Test:** `test_BC_1_05_001_idmz_bypasses_le_total()` (Kani)

### AC-003 (traces to BC-1.05.001 postcondition 3 ‚Äî distinct_violating_flows ‚â§ total_flows)
`distinct_violating_flows ‚â§ total_flows` holds.
- **Test:** `test_BC_1_05_001_distinct_violating_le_total()` (Kani)

### AC-004 (traces to BC-1.05.001 postcondition 4 ‚Äî skipped excluded from total)
`skipped` is a separate counter passed to `aggregate`. It does NOT contribute to `total_flows`.
- **Test:** `test_BC_1_05_001_skipped_not_in_total_flows()`

### AC-005 (traces to BC-1.05.001 postcondition 5 ‚Äî external_endpoints separate)
`external_endpoints` (count of flows with ‚â•1 EXTERNAL endpoint) is a separate diagnostic counter, NOT included in the DI-015 identity. A flow with an EXTERNAL src is still counted in its VerdictKind tally.
- **Test:** `test_BC_1_05_001_external_endpoints_diagnostic_only()`

### AC-006 (traces to BC-1.05.004 postcondition ‚Äî overflow abort)
When any u64 tally increment would overflow, `aggregator::aggregate` returns `Err(SysError::TallyOverflow)`. It does NOT silently wrap. The pipeline propagates this error to exit 2.
- **Test:** `test_BC_1_05_004_tally_overflow_returns_err()`

### AC-007 (traces to BC-1.05.005 postcondition ‚Äî empty input)
With zero input flows (`skipped = 0`), `aggregate` returns a `ConformanceResult` where all tallies are 0, `violations` is empty, and `total_flows == 0`.
- **Test:** `test_BC_1_05_005_empty_input_all_zero()`

### AC-008 (traces to BC-1.05.001 invariant 1 ‚Äî identity holds for all inputs)
The DI-015 accounting identity holds for the empty-input case: `0 == 0+0+0+0+0`.
- **Test:** Covered by AC-007 + `test_BC_1_05_001_di015_identity_holds()` with empty iterator

### AC-009 (traces to BC-1.05.004 ‚Äî u64 arithmetic; no usize)
All tally fields are `u64`. No `usize` counter is used in the hot path (per DI-015 and SS-02-data-model.md).
- **Test:** Compile-time: `ConformanceResult` fields are `u64`; `checked_add` used throughout
