# VP-006: Tally Overflow Detection — No Silent Wrap (FM-009)

## Property Statement

`aggregator::increment_tally(current: u64, increment: u64) -> Result<u64, SysError>`
uses `checked_add`. If `current + increment` would exceed `u64::MAX`, the function
returns `Err(SysError::TallyOverflow)`. It never silently wraps (i.e., never uses
`wrapping_add`), and never panics.

This is a defense-in-depth backstop: the FM-008 ingest cap (`max_flows`) makes overflow
unreachable in practice, but the checked arithmetic is proven correct regardless.

## Source Contract

- **BC:** BC-1.05.004 — u64 tally overflow: checked arithmetic aborts with exit 2; never silent-wraps
- **Postcondition/Invariant:** FM-009 (checked arithmetic on counters)

| Method | Tool | Bounded? | Coverage |

## Proof Harness Skeleton

```rust
#[cfg(kani)]
#[kani::proof]
fn vp006_tally_no_silent_wrap() {
    let current: u64 = kani::any();
    let increment: u64 = kani::any();

    let result = aggregator::increment_tally(current, increment);

    match result {
        Ok(new_val) => {
            // Must equal the mathematical sum (no wrap)
            assert_eq!(new_val, current + increment);
            // Must not have overflowed
            assert!(current.checked_add(increment).is_some());
        }
        Err(SysError::TallyOverflow) => {
            // Must only error when overflow would have occurred
            assert!(current.checked_add(increment).is_none());
        }
    }

    // The function never panics — Kani checks this automatically
}
```
