# VP-002: Longest-Prefix Totality and Uniqueness (DI-004)

## Property Statement

For a valid `PrefixIndex` (sorted descending by prefix length, tie-free within each IP family),
`resolver::resolve` returns the entry with the strictly longest matching prefix. If multiple
entries match the same IP, the one with the greater prefix length is always selected. This
is unique given the tie-free invariant enforced by `validator`.

Formally: if `entry_a.matches(ip) ∧ entry_b.matches(ip) ∧ entry_a.prefix_len > entry_b.prefix_len`
then `resolve(index, ip).zone_id == entry_a.zone_id`.

## Source Contract

- **BC:** BC-1.03.001 — longest-prefix match is total and unique
- **Postcondition/Invariant:** DI-004 (deterministic overlap resolution)
- **Also covers:** BC-1.01.005 (tie rejection at validation time)

## Proof Harness Skeleton

```rust
#[cfg(kani)]
#[kani::proof]
#[kani::unwind(64)]
fn vp002_longest_prefix_wins() {
    // Construct a two-entry index where entry_a is longer prefix than entry_b
    // and both match a common IP
    let ip: Ipv4Addr = kani::any();

    // entry_a: /24 (longer)
    // entry_b: /16 (shorter, covering the same range)
    // Precondition: ip is in both prefixes
    let net_a = Ipv4Net::new(kani::any(), 24).unwrap();
    let net_b = Ipv4Net::new(kani::any(), 16).unwrap();
    kani::assume(net_a.contains(&ip) && net_b.contains(&ip));
    kani::assume(net_b.network() == net_a.network().mask(16)); // net_b covers net_a

    let index = PrefixIndex::from_pairs_v4(vec![
        (net_a, zone_id("zone-a")),
        (net_b, zone_id("zone-b")),
    ]);
    kani::assume(index.is_valid());

    let result = resolver::resolve(&index, IpAddr::V4(ip));

    // Longest prefix (zone-a, /24) must win
    assert_eq!(result.zone_id, zone_id("zone-a"));
    assert!(matches!(result.match_kind, MatchKind::Explicit { prefix_len: 24 }));
}
```
