# Domain Invariants (Business Rules)

> These are the rules that must always hold. They seed the PRD behavioral contracts and the
> Kani/property tests. Several are candidates for formal proof (flagged ⊢).

| ID | Rule | Scope | Violation Behavior |
|----|------|-------|--------------------|
| DI-001 | **Deny-by-default.** Any flow not explicitly permitted (by a matching conduit) or intra-zone is a Violation. | classification | Verdict `NoMatchingConduit` |
| DI-002 | **Intra-zone is allowed.** A flow whose two endpoints resolve to the *same* zone is `IntraZone` — conduits do not govern it. The reserved `EXTERNAL` zone is a **single zone**, so two endpoints both resolving to `EXTERNAL` (e.g. `8.8.8.8`→`1.1.1.1`) satisfy "same zone" → `IntraZone` (Internet↔Internet transit is out of scope for managed segmentation; DEC-032). | classification | Verdict `IntraZone`; never a violation |
| DI-003 | **Total endpoint resolution.** Every flow endpoint resolves to exactly one outcome: a single zone, the implicit `EXTERNAL` zone, or (for a multicast/broadcast destination) the `MulticastBroadcast` match kind. ⊢ | zone resolution | Never silently dropped/allowed |
| DI-004 | **Deterministic overlap resolution.** When an IP matches multiple declared matchers, the **longest prefix** wins. `EXTERNAL` is **not** a `0.0.0.0/0` matcher — it is the result when the declared matcher set yields no match (a fallback, outside longest-prefix). ⊢ | zone resolution | Longest-prefix selection is total & unique given a valid policy |
| DI-005 | **Implicit EXTERNAL zone.** IPs matching no declared matcher resolve to the reserved `EXTERNAL` zone (fixed `purdue_level = L5`, a sentinel); flows to/from it are governed by conduits like any zone (deny-by-default), never by the IDMZ rule. | zone resolution | Governed normally; reported with `match = ImplicitExternal` |
| DI-006 | **IDMZ no-bypass (single-flow, additive).** `idmz_bypass = true` for a flow **iff** one endpoint resolves to a *managed* (non-EXTERNAL) zone whose level is ≤ L3 and the other to a *managed* zone whose level is ≥ L4, and **neither endpoint is in the IDMZ** (L3.5). **"Managed" vs EXTERNAL is decided by zone identity** (the reserved `EXTERNAL` zone id), **not by `purdue_level`** — a declared zone may legitimately be L5 and is still "managed". It is an **independent additive finding**, NOT a `VerdictKind` — raised in addition to (and tallied separately from) the conduit verdict. Per single flow only; multi-hop transit is **out of scope**. EXTERNAL endpoints are excluded (handled by conduit matching); if **either endpoint is `MulticastBroadcast`, `idmz_bypass = false`** (this is the *single* exception to `idmz_bypass`'s otherwise verdict-independent evaluation — implementers must special-case it). See the truth table below. ⊢ | architecture check | Record `idmz_bypass`; emit a `Violation{IdmzBypass}` regardless of the conduit verdict — a flow may be both `Allowed` and an IdmzBypass |
| DI-007 | **Conduit directionality.** `from→to` permits only flows whose **initiator (src)** is in `from` and `dst` is in `to`. A `Bidirectional` conduit permits initiation from either side. | conduit matching | Reverse-initiated flow on a `Forward` conduit → `WrongDirection` |
| DI-008 | **Service provenance honesty.** `service_source` is always set; port-derived services are `PortHeuristic` (or `Unknown`) and never reported as authoritative. | normalization/report | Reports must show provenance; heuristic services flagged |
| DI-009 | **Determinism.** Identical inputs (policy + flows) always yield identical verdicts and byte-stable output. ⊢ | whole pipeline | Output ordered by the **total-order key** `(ts, src_ip, src_port, dst_ip, dst_port, proto)` at full `ts` precision, with `flow_index` as the final stable tiebreaker — unique ordering over any flow multiset. `Other(String)`/`Other(u8)` payloads (Service/Proto/ConnState) are preserved **verbatim** (no Unicode/case normalization) and compared by exact bytes; they are byte-stable iff the input is |
| DI-010 | **Policy validity.** Zone ids unique; conduit endpoints exist; `purdue_level` ∈ the defined set; no membership **ties**; `EXTERNAL` not redefined. A **tie** = two `AssetMatcher`s, **in the same IP family**, that match a common address with **equal prefix length**. Disjoint same-length CIDRs (e.g. `10.0.0.0/24` vs `10.0.1.0/24`) are **not** a tie and are legal (DEC-022). Also validates **token legality** — legal `direction` tokens are exactly `{forward, bidirectional, unidirectional(=forward alias)}`; legal `proto` tokens `{tcp, udp, icmp, other:<u8>}`; any unrecognized `direction`/`proto` or malformed `PortSet` → policy error, no permissive defaulting — **duplicate YAML mapping keys** (load error — no silent last-wins; diverges from serde_yaml defaults), and **rejects a declared `0.0.0.0/0` or `::/0` member** (a family catch-all would shadow the implicit EXTERNAL zone; DEC-029). | policy load | Fail fast, non-zero exit, precise diagnostic; no flow processing |
| DI-011 | **No partial state.** A policy either fully loads & validates or is rejected; the engine never runs on a partially-parsed policy. | policy load | Reject whole policy |
| DI-012 | **Read-only / offline.** zonewarden never opens a network socket, never mutates inputs, never enforces. Pure function of its file inputs. ⊢ | whole tool | Any network/enforcement action is out of scope by construction |
| DI-013 | **Ingest resilience asymmetry.** A malformed *flow* record is skipped and counted (`skipped`), never aborting the run; a malformed/invalid *policy* always aborts (DI-010/011). "Fail loud on policy, degrade gracefully on flows." `flow_index` counts **only successfully-normalized flows** (dense, gap-free `0..n`); skipped records do not consume an index. | flow ingest / policy load | Skipped flows tallied + surfaced; a clean run with `skipped > 0` exits 0 **with a mandatory warning** unless `--fail-on-skipped` is set (DEC-024) |
| DI-014 | **Conduit matching is any-match (union).** A flow is `Allowed` if **at least one** conduit permits its (zone-pair, proto, port, direction). Multiple/overlapping/duplicate conduits between the same zone-pair are legal and unioned — never a load-time error. | conduit matching | A flow allowed by ≥1 conduit is `Allowed`; a flow allowed by *no* conduit (and not intra-zone / multicast) is `NoMatchingConduit` (DI-001) |
| DI-015 | **Verdict totality & accounting.** Every resolved flow receives **exactly one** `VerdictKind` (the conduit/zone classification). Accounting over that kind: `total_flows == intra_zone + allowed + no_matching_conduit + wrong_direction + multicast_exempt`. `idmz_bypasses` is tallied **separately** and may overlap `allowed`/etc. `distinct_violating_flows` de-dups by `flow_index`, and **`distinct_violating_flows ≤ total_flows`** and **`idmz_bypasses ≤ total_flows`** always hold. `skipped` is counted separately and excluded from `total_flows`. ⊢ | classification / aggregation | A property test asserts the accounting identity and both ≤ bounds for any input |
| DI-016 | **Multicast/broadcast exemption + verdict precedence.** A flow whose **destination** IP is a multicast or broadcast address resolves to `MatchKind::MulticastBroadcast` → verdict `MulticastExempt` — never `NoMatchingConduit`. **Verdict precedence is the ST-6 order: `MulticastExempt` short-circuits *before* zone-pair classification** (a multicast dst wins over `IntraZone`). **Detection is a two-step destination resolution (no circularity):** (1) **family-wide test, needs no zone** — `dst ∈ 224.0.0.0/4` (v4 multicast), `dst == 255.255.255.255` (limited broadcast), or `dst ∈ ff00::/8` (v6 multicast) → `MulticastBroadcast` immediately; (2) otherwise resolve `dst` to its longest-prefix zone (DI-004), then **if the matched zone's IPv4 prefix length is ≤ 30 AND `dst` equals that zone's directed-broadcast (all-ones host) address, override the `Explicit` MatchKind to `MulticastBroadcast`** (override has precedence over `Explicit`). **The /31 and /32 cases are excluded** — they have no usable broadcast address, so a single-host (/32) or point-to-point (/31) member is never silently exempted (closes the silent-allow vector R-002). **Step 2 is IPv4-only** — IPv6 has no broadcast, so a v6 dst is only exempt via the step-1 `ff00::/8` test and never via this override. An `EXTERNAL` dst (no matched CIDR) cannot be a directed broadcast. The override is the only case where resolution runs before the exemption verdict. | classification | `MulticastBroadcast` → `MulticastExempt`; reported, not a violation; `idmz_bypass = false` (DI-006) |
| DI-017 | **Severity grading by conn_state.** A violation's `severity` is `Attempted` **iff** the flow's `ConnState` is in the **Attempted bucket** as defined in entities.md (`ConnState`); otherwise `Established` (covers the Established bucket, `Other(_)`, and absent `conn_state` — conservative). entities.md `ConnState` is the single source of bucket membership (no per-state list duplicated here). | classification/report | Violations carry `severity`; blocked attempts are not over-reported as confirmed breaches |
| DI-018 | **Stable policy digest (canonicalization defined).** `policy_digest` = `SHA-256` (lowercase hex) over the **canonical serialization** of the Policy model, defined as: **canonical JSON, UTF-8, object keys sorted lexicographically**; zones sorted by `id`; conduits **de-duplicated** then sorted by `(from_zone, to_zone, proto, direction, normalized_ports)`; each `PortSet` normalized per DI-020; `None`/absent Option fields **omitted** entirely; `Zone.name` and `sl_t` **are** digested (numbers as JSON integers, no leading zeros). **Enum sort/serialization tokens are fixed:** `Proto` → `"tcp"`/`"udp"`/`"icmp"`/`"other:<u8>"`; `Direction` → `"forward"`/`"bidirectional"`; `PortSet` → `"any"` (sorts last) or `"[lo-hi,…]"`. **Collation = byte-wise over UTF-8** (not codepoint), applied consistently to keys and `Other(String)` payloads. Two model-identical policies (whitespace/comment/key-order/duplicate-conduit differences) thus produce the **same** digest. ⊢ | report | Identical model → identical digest; supports the edit-and-rerun demo determinism |
| DI-019 | **Deterministic warning model.** Warnings are emitted to **stderr** and collected in `ConformanceResult.warnings: Vec<String>`, in a stable, deterministic order (ordering governed by DI-009's determinism guarantee; DI-019 is a distinct invariant, not subsumed by DI-009). Warnings **never** change the exit code (only `--fail-on-skipped` does). | whole pipeline | Warning content/order is reproducible; the JSON report's determinism is unaffected by stderr capture |
| DI-020 | **PortSet canonical form.** A `PortSet` normalizes to a sorted list of **non-overlapping, non-adjacent** inclusive `[lo,hi]` ranges (singletons are `[p,p]`; adjacent ranges/ports **coalesce**, e.g. `500-502`+`503` → `500-503`). `Any` is a **distinct sentinel** and is **never** folded with `0-65535` (they differ semantically: `Any` matches portless flows, `0-65535` does not — DEC-021). Used by matching (DI-014) and the digest (DI-018). ⊢ | normalization / digest | Unique representation per port-set value; deterministic across implementations |

## DI-006 Truth Table (IDMZ bypass)

> Endpoint A vs Endpoint B by zone kind/level. "managed" = a declared, non-EXTERNAL zone.
> `idmz_bypass = true` only for managed ≤L3 ↔ managed ≥L4 with neither endpoint in the IDMZ.

| Endpoint A | Endpoint B | idmz_bypass? | Why |
|------------|------------|--------------|-----|
| managed L1 (≤L3) | managed L4 (≥L4) | **yes** | direct OT↔IT, no IDMZ endpoint |
| managed L1 (≤L3) | managed L5 (≥L4) | **yes** | declared L5 is a managed endpoint |
| managed L1 (≤L3) | **EXTERNAL** (L5 sentinel) | no | EXTERNAL excluded; governed by conduit (DI-005) |
| managed L2 | IDMZ (L3.5) | no | one endpoint IS the IDMZ |
| IDMZ (L3.5) | managed L4 | no | one endpoint IS the IDMZ |
| IDMZ | IDMZ | no | neither side is ≤L3 or ≥L4 |
| managed L2 | managed L3 | no | both ≤L3 (no IT side) |
| managed L4 | managed L5 | no | both ≥L4 (no OT side) |
| any | MulticastBroadcast (dst) | no | multicast/broadcast dst → MulticastExempt; idmz_bypass forced false (DI-016) |
