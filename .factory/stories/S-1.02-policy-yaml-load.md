# S-1.02: Policy YAML Load + Type Model

## Narrative
- **As a** zonewarden user
- **I want to** load a YAML segmentation policy file into the internal `Policy` model
- **So that** the tool has a structured, immutable representation of zones and conduits to validate and run the pipeline against

## Acceptance Criteria

### AC-001 (traces to BC-1.01.001 postcondition 1)
`policy::load(path)` returns a `Policy` value with all zones and conduits populated from a valid YAML file. A minimal 2-zone + 1-conduit policy loads without error.
- **Test:** `test_BC_1_01_001_valid_minimal_policy_loads()`

### AC-002 (traces to BC-1.01.001 postcondition 2)
Each `Zone` in the returned `Policy` has: `id` (non-empty string), `name`, a valid `PurdueLevel` variant, optional `sl_t`, and `members` (may be empty vec).
- **Test:** `test_BC_1_01_001_zone_fields_populated()`

### AC-003 (traces to BC-1.01.001 postcondition 3)
Each `Conduit` has: `from_zone`, `to_zone`, `direction` (Forward or Bidirectional), `proto` (Tcp/Udp/Icmp/Other(u8)), and `ports: PortSet`.
- **Test:** `test_BC_1_01_001_conduit_fields_populated()`

### AC-004 (traces to BC-1.01.001 postcondition 4)
`PortSet` values parsed from YAML are in canonical form per DI-020 (delegated to `portset::canonicalize`). Adjacent port list `[500, 501]` is stored as `[500-501]`.
- **Test:** `test_BC_1_01_001_portset_canonical_at_load()`

### AC-005 (traces to BC-1.01.001 edge case EC-003)
`direction: unidirectional` is accepted as an alias and normalized to `Direction::Forward`.
- **Test:** `test_BC_1_01_001_unidirectional_alias_normalized()`

### AC-006 (traces to BC-1.01.002 postcondition — error path)
`policy::load` returns `Err(E-POL-001)` on an empty YAML file and `Err(E-POL-002)` when a required zone field (`purdue_level`) is missing. The error message includes the file path and the offending location.
- **Test:** `test_BC_1_01_002_empty_file_returns_pol_001()`, `test_BC_1_01_002_missing_field_returns_pol_002()`

### AC-007 (traces to BC-1.01.003 postcondition — duplicate key rejection)
`policy::load` returns `Err(E-POL-004)` when the YAML file contains a duplicate mapping key (e.g., two `id:` fields on the same zone object). Uses `serde_norway` strict mode.
- **Test:** `test_BC_1_01_003_duplicate_key_rejected()`

### AC-008 (traces to BC-1.01.001 invariant 1)
The returned `Policy` value is immutable (Rust ownership semantics; no interior mutability). The raw YAML bytes are not retained after parsing.
- **Test:** Compile-time: `Policy` does not implement `DerefMut` or expose `&mut` fields

### AC-009 (traces to BC-1.01.001 invariant 2)
Parsing is deterministic: calling `policy::load` twice with the same bytes yields identical `Policy` values.
- **Test:** `test_BC_1_01_001_parse_is_deterministic()` (proptest round-trip)

## Architecture Mapping

| Component | Module | Pure/Effectful |
|-----------|--------|---------------|
| `policy` (YAML load) | `zonewarden/src/policy.rs` | Effectful (file I/O + serde) |
| `Policy` type | `zonewarden-core/src/types.rs` | Pure (already defined in S-1.01) |
| `portset` | `zonewarden-core/src/portset.rs` | Pure (called at load to canonicalize) |

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | YAML with comments and extra whitespace | Parses successfully |
| EC-002 | Policy with zero conduits | Parses; `conduits` is empty vec |
| EC-003 | `direction: unidirectional` on conduit | Normalized to `Direction::Forward` |
| EC-004 | `sl_t: 3` (scalar) | `SlTarget { overall: 3, fr_vector: None }` |
| EC-005 | `sl_t` as 7-element FR vector | `SlTarget { overall, fr_vector: Some([...]) }` |
| EC-006 | `ports: any` (lowercase) | Parsed as `PortSet::Any` |
| EC-007 | `[500, 501]` adjacent ports | Coalesced to `[500-501]` by portset::canonicalize |
| EC-008 | File not found at `--policy` path | `Err(E-IO-001)` with path in message |
| EC-009 | File exists but permission denied | `Err(E-IO-002)` |

## Architecture Compliance Rules

| Rule | Source | Enforcement |
|------|--------|-------------|
| YAML parser MUST be `serde_norway` (not `serde_yaml`) | ADR-005 | `Cargo.toml` dependency; `serde_yaml` must not appear |
| `policy` module lives in the `zonewarden` binary crate (effectful shell) | ADR-002 | Module path: `zonewarden/src/policy.rs` |
| All `PolicyError` variants map to exit code 2 | ADR-006, BC-1.06.001 | Exit code mapping enforced in `cli::main` (not yet, but error types established here) |
| No `.unwrap()` in `policy::load` hot path | SS-03-cli-adapter.md | `#[deny(clippy::unwrap_used)]` |
| Raw YAML bytes not retained post-parse | BC-1.01.001 postcondition 6 | No `String` field on `Policy` storing raw YAML |
