# S-1.01: Workspace Scaffold + PortSet Canonical Form

## Narrative
- **As a** zonewarden implementer
- **I want to** initialize the Cargo workspace (pure `zonewarden-core` lib + thin `zonewarden` binary) with the core data types and the `portset` module implementing DI-020 canonical form
- **So that** every downstream module has a compilable, correctly-typed foundation to build on, with the critical PortSet matching invariant formally verified

## Acceptance Criteria

### AC-001 (traces to BC-1.01.009 postcondition 1)
The `portset::canonicalize` function returns a `PortSet` in canonical form: sorted, non-overlapping, non-adjacent inclusive `[lo, hi]` ranges. Given `[500, 501, 502]`, it returns a single range `[500-502]`.
- **Test:** `test_BC_1_01_009_adjacent_singletons_coalesced()`

### AC-002 (traces to BC-1.01.009 postcondition 2)
`PortSet::Any` is preserved as `Any` and is never converted to or from `[0, 65535]`. `canonicalize(Any) == Any`; `canonicalize([0-65535]) != Any`.
- **Test:** `test_BC_1_01_009_any_sentinel_distinct_from_full_range()`

### AC-003 (traces to BC-1.01.009 postcondition 3)
Two `PortSet` values that are equivalent as port-number sets produce equal canonical forms after normalization. Given `[100-200, 150-250, 300]`, result is `[100-250, 300-300]`.
- **Test:** `test_BC_1_01_009_overlapping_ranges_coalesced()`

### AC-004 (traces to BC-1.01.009 invariant 2)
No two ranges in the canonical form are adjacent (`lo2 > hi1 + 1` must hold). Adjacent ranges `[500-502, 503-505]` are coalesced to `[500-505]`.
- **Test:** `test_BC_1_01_009_adjacent_ranges_coalesced()`

### AC-005 (traces to BC-1.01.009 invariant 4)
The canonical form is unique: there is exactly one canonical representation for each port set value. Normalization is idempotent: `canonicalize(canonicalize(x)) == canonicalize(x)`.
- **Test:** `test_BC_1_01_009_idempotent_normalization()` (proptest)

### AC-006 (traces to BC-1.01.009 postcondition 4)
Port ranges with `lo > hi` are rejected during construction (returns `Err`); they do not produce a canonical form.
- **Test:** `test_BC_1_01_009_inverted_range_rejected()`

### AC-007 (workspace scaffold)
The Cargo workspace compiles cleanly with `cargo check` on both `zonewarden-core` and `zonewarden` crates. The `zonewarden-core` crate declares `#![forbid(unsafe_code)]` and `#![deny(clippy::unwrap_used)]` on hot-path modules.
- **Test:** `cargo check --workspace` in CI; `cargo clippy --workspace -- -D warnings`
