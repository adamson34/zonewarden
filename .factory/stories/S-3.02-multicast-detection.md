# S-3.02: Multicast + Directed-Broadcast Detection

## Narrative
- **As a** zonewarden pipeline
- **I want to** detect multicast/broadcast destination addresses (including directed broadcasts within zone CIDRs) before zone resolution runs
- **So that** these flows are classified as `MulticastExempt` rather than generating false `NoMatchingConduit` violations at scale

## Acceptance Criteria

### AC-001 (traces to BC-1.03.003 postcondition)
`multicast::classify_dst(dst_ip, prefix_index)` returns `DstKind::MulticastBroadcast` for any IPv4 address in `224.0.0.0/4` (multicast range).
- **Test:** `test_BC_1_03_003_ipv4_multicast_range_detected()`

### AC-002 (traces to BC-1.03.003 postcondition)
`multicast::classify_dst` returns `DstKind::MulticastBroadcast` for the IPv4 limited broadcast address `255.255.255.255`.
- **Test:** `test_BC_1_03_003_ipv4_limited_broadcast_detected()`

### AC-003 (traces to BC-1.03.003 postcondition)
`multicast::classify_dst` returns `DstKind::MulticastBroadcast` for any IPv6 address in `ff00::/8` (IPv6 multicast range).
- **Test:** `test_BC_1_03_003_ipv6_multicast_range_detected()`

### AC-004 (traces to BC-1.03.004 postcondition)
`multicast::classify_dst` returns `DstKind::MulticastBroadcast` for a directed-broadcast address (all-ones host bits) of a zone CIDR with prefix length â‰¤ 30. For zone `10.0.1.0/24`, destination `10.0.1.255` is a directed broadcast.
- **Test:** `test_BC_1_03_004_directed_broadcast_detected_le30()`

### AC-005 (traces to BC-1.03.004 edge case â€” /31 and /32 exclusion)
For a zone with prefix length 31 or 32, the all-ones host address is NOT treated as a directed broadcast (no directed-broadcast concept for /31 or /32). A `/31` zone `10.0.0.0/31` does NOT flag `10.0.0.1` as directed-broadcast.
- **Test:** `test_BC_1_03_004_directed_broadcast_not_applied_to_31_32()`

### AC-006 (traces to BC-1.03.003 postcondition â€” multicast runs first)
`multicast::classify_dst` short-circuits before zone resolution. A multicast destination address that also happens to fall within a declared zone CIDR is still classified `MulticastBroadcast` (the zone match is irrelevant).
- **Test:** `test_BC_1_03_003_multicast_in_zone_cidr_still_exempt()`

### AC-007 (traces to BC-1.03.004 â€” IPv4-only constraint)
Directed-broadcast detection applies only to IPv4 addresses. IPv6 directed broadcasts are not a protocol concept; an IPv6 all-ones address is classified via the IPv6 multicast check only (ff00::/8).
- **Test:** `test_BC_1_03_004_directed_broadcast_ipv4_only()`
