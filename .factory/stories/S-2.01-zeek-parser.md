# S-2.01: Zeek conn.log Parser + Flow Normalization

## Narrative
- **As a** zonewarden pipeline
- **I want to** stream-parse a Zeek `conn.log` file line-by-line into normalized `Flow` records
- **So that** the classifier receives a well-typed, canonicalized representation of each observed network connection

## Acceptance Criteria

### AC-001 (traces to BC-1.02.001 postcondition 1)
A valid Zeek TSV data line is parsed into a `Flow` with all required fields: `flow_index`, `ts` (nanosecond UTC), `src_ip` (id.orig_h), `src_port` (id.orig_p — None if 0 or `-`), `dst_ip` (id.resp_h), `dst_port`, `proto`, `conn_state` (None if `-`).
- **Test:** `test_BC_1_02_001_valid_line_produces_flow()`

### AC-002 (traces to BC-1.02.001 invariant 4)
`flow_index` values assigned to successfully-normalized flows are dense and gap-free (0, 1, 2, ...), counting only successful flows — not skipped header lines or malformed lines.
- **Test:** `test_BC_1_02_001_flow_index_dense_gap_free()` (proptest)

### AC-003 (traces to BC-1.02.001 edge case EC-001)
Lines starting with `#` (Zeek header/comment lines: `#fields`, `#separator`, `#types`, etc.) are silently skipped. They do NOT increment the `skipped` counter and do NOT affect `flow_index`.
- **Test:** `test_BC_1_02_001_header_lines_silently_skipped()`

### AC-004 (traces to BC-1.02.002 postcondition)
Malformed data lines (missing required field, wrong column count, unparseable timestamp) are skipped and the `skipped` counter is incremented by 1. The parse run continues to the next line — no abort.
- **Test:** `test_BC_1_02_002_malformed_line_skipped_counted()`

### AC-005 (traces to BC-1.02.002 invariant — never abort)
A file containing 100% malformed data lines produces `skipped = N` and zero Flow records, but `policy::load` does not return an error — the run completes with exit 0 (assuming no violations in zero flows).
- **Test:** `test_BC_1_02_002_all_malformed_never_aborts()`

### AC-006 (traces to BC-1.02.003 postcondition)
A flow line where `id.orig_h` or `id.resp_h` is `0.0.0.0` or `::` is skipped with a warning written to the `warnings` vec, and `skipped` is incremented. It does NOT produce a Flow.
- **Test:** `test_BC_1_02_003_unspecified_src_skipped_with_warning()`, `test_BC_1_02_003_unspecified_dst_skipped_with_warning()`

### AC-007 (traces to BC-1.02.005 postcondition)
IPv4-mapped IPv6 addresses in the form `::ffff:a.b.c.d` in `id.orig_h` or `id.resp_h` are canonicalized to plain IPv4 addresses before the Flow is produced.
- **Test:** `test_BC_1_02_005_ipv4_mapped_canonicalized()`

### AC-008 (traces to BC-1.02.001 edge case EC-007)
Lines with Windows-style CRLF line endings (`\r\n`) are parsed correctly; the trailing `\r` does not corrupt field parsing.
- **Test:** `test_BC_1_02_001_crlf_line_endings_handled()`

| EC-003 | `::ffff:10.0.1.5` as src | Canonicalized to `10.0.1.5` |
| EC-004 | `0.0.0.0` as src | Skipped with E-FLW-002 warning |
| EC-005 | `::` as dst | Skipped with E-FLW-002 warning |
| EC-006 | Header line `#fields ts id.orig_h ...` | Silently skipped; skipped not incremented |
| EC-007 | Truncated line (missing proto field) | Skipped; `skipped` incremented; E-FLW-001 |
| EC-008 | CRLF line endings | Parsed correctly |
| EC-009 | Empty file | Zero flows; zero skipped; run completes |

## Architecture Compliance Rules

| Rule | Source | Enforcement |
|------|--------|-------------|
| `zeek_adapter` streams line-by-line; never materializes full file | ADR-001, OQ-003 | BufReader<File> iterator; no `read_to_string` |
| `flow_index` counts only successfully-normalized flows (dense, gap-free) | BC-1.02.001 invariant 4, DI-013 | Counter incremented only on `Ok(flow)` path |
| `RealitySource` trait defined in `zonewarden-core` (pure side) | ADR-002, SS-02-data-model.md | Trait in `zonewarden-core/src/lib.rs` |
| No network I/O; reads from file only | BC-1.06.007, DI-012 | No `std::net` use in adapter |
| Header lines (starting with `#`) do NOT increment `skipped` | BC-1.02.001 edge case EC-001 | Explicit check before malformed-line path |
