# S-4.01: Severity Grading (conn_state → Bucket)

## Narrative
- **As a** zonewarden violation reporter
- **I want to** assign an `Established` or `Attempted` severity bucket to each violation based on the flow's `conn_state`
- **So that** the report faithfully grades the real-world impact of each detected violation

## Acceptance Criteria

### AC-001 (traces to BC-1.04.009 postcondition 1)
`severity::grade(conn_state)` returns `Severity::Established` for `ConnState::Established`. This covers Zeek states: `SF`, `S2`, `S3`, `RSTO`, `RSTR`, `OTH`.
- **Test:** `test_BC_1_04_009_established_states_map_to_established()`

### AC-002 (traces to BC-1.04.009 postcondition 1)
`severity::grade(conn_state)` returns `Severity::Attempted` for `ConnState::Attempted`. This covers Zeek states: `S0`, `S1`, `REJ`, `RSTOS0`, `RSTRH`, `SH`, `SHR`.
- **Test:** `test_BC_1_04_009_attempted_states_map_to_attempted()`

### AC-003 (traces to BC-1.04.009 postcondition 2 — conservative default)
`severity::grade(None)` (conn_state absent) returns `Severity::Established` (conservative default — assume completed).
- **Test:** `test_BC_1_04_009_none_conn_state_defaults_to_established()`

### AC-004 (traces to BC-1.04.009 postcondition — full 13-state table per OQ-001)
The complete OQ-001 conn_state table is implemented:
- `S0` → Attempted; `S1` → Attempted; `S2` → Established; `S3` → Established; `SF` → Established
- `REJ` → Attempted; `RSTO` → Established; `RSTR` → Established; `RSTOS0` → Attempted; `RSTRH` → Attempted
- `SH` → Attempted; `SHR` → Attempted; `OTH` → Established
- **Test:** `test_BC_1_04_009_full_13_state_table_correct()` (parameterized)

### AC-005 (traces to BC-1.04.009 invariant — DI-017 single source)
The conn_state → bucket mapping has exactly ONE definition in the codebase (DI-017 compliance). It lives in `severity.rs` and is not duplicated elsewhere.
- **Test:** Enforced via code review + grep-based CI check

| EC-001 | `conn_state: None` (absent/`-`) | `Severity::Established` (conservative default) |

| EC-003 | `RSTO` (reset by originator after established) | `Severity::Established` |

| EC-006 | `SHR` (SYN + SYN/ACK + RST) | `Severity::Attempted` |
| EC-007 | `RSTOS0` (SYN then RST before response) | `Severity::Attempted` |
