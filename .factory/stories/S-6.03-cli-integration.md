# S-6.03: CLI — Argument Parsing, Exit Codes, Full Pipeline Integration

## Narrative
- **As a** zonewarden user
- **I want to** run `zonewarden --policy <PATH> --flows <PATH> [options]` and receive the correct exit code and formatted output
- **So that** the tool is fully integrated end-to-end and usable in CI pipelines and command-line assessments

## Acceptance Criteria

### AC-001 (traces to BC-1.06.001 postcondition 1 — exit 0)
`zonewarden --policy valid.yaml --flows conformant.log` exits 0 when `distinct_violating_flows == 0 && idmz_bypasses == 0`.
- **Test:** `test_BC_1_06_001_exit_0_conformant()` (integration test)

### AC-002 (traces to BC-1.06.001 postcondition 2 — exit 1)
`zonewarden --policy valid.yaml --flows violating.log` exits 1 when `distinct_violating_flows > 0 || idmz_bypasses > 0`.
- **Test:** `test_BC_1_06_001_exit_1_violations()` (integration test)

### AC-003 (traces to BC-1.06.001 postcondition 3 — exit 2 on policy error)
`zonewarden --policy invalid.yaml --flows any.log` exits 2 with a diagnostic to stderr. No report is written.
- **Test:** `test_BC_1_06_001_exit_2_policy_error()` (integration test)

### AC-004 (traces to BC-1.06.001 postcondition 3 — exit 2 on missing file)
`zonewarden --policy missing.yaml --flows any.log` exits 2 with `E-IO-001` error message to stderr.
- **Test:** `test_BC_1_06_001_exit_2_missing_policy_file()`

### AC-005 (traces to BC-1.06.006 postcondition — --fail-on-skipped)
With `--fail-on-skipped`: if `skipped > 0` and would otherwise exit 0, exits 1 instead. If there are also violations, still exits 1 (the exit code does not change from 1).
- **Test:** `test_BC_1_06_006_fail_on_skipped_upgrades_exit_0_to_1()`

### AC-006 (traces to BC-1.06.006 invariant — skipped warning regardless)
Regardless of `--fail-on-skipped`, warnings about skipped flows are always emitted to stderr.
- **Test:** `test_BC_1_06_006_skipped_warning_always_emitted()`

### AC-007 (traces to BC-1.06.007 invariant — no network socket)
The full pipeline integration test verifies no network socket is opened during a complete run. Structural enforcement: `zonewarden-core` has no `std::net` import; binary crate uses `cargo deny`.
- **Test:** `cargo deny check` in CI with `deny = ["net"]`; `grep -rn "TcpStream\|UdpSocket\|std::net" zonewarden/src/` returns empty

### AC-008 (traces to BC-1.06.001 postcondition 3 — exit 2 on cap breach)
`zonewarden --policy valid.yaml --flows huge.log --max-flows 5` exits 2 when flow count exceeds 5. Stderr includes cap value and count.
- **Test:** `test_BC_1_06_001_exit_2_cap_breach()`

### AC-009 (traces to BC-1.06.001 — full pipeline end-to-end)
A complete integration test: valid policy (2 zones, 1 conduit), 5 flows (3 allowed, 1 violation, 1 intra-zone), format=json. Output JSON passes schema validation; exit 1 (violation present).
- **Test:** `test_full_pipeline_end_to_end()`

### AC-010 (traces to BC-1.06.006 — --max-flows 0 rejected)
`zonewarden --policy any.yaml --flows any.log --max-flows 0` exits 2 immediately with `E-SYS-002` before opening any file.
- **Test:** `test_BC_1_06_006_max_flows_zero_rejected()`

### AC-011 (traces to BC-1.06.001 — IDMZ bypass only exits 1)
If `distinct_violating_flows == 0` but `idmz_bypasses > 0`, exit code is 1 (not 0). IDMZ bypass alone triggers exit 1.
- **Test:** `test_BC_1_06_001_exit_1_idmz_bypass_only()`

## Tasks

1. [ ] Write failing integration tests covering AC-001 through AC-011 (using `std::process::Command` to run the binary)
2. [ ] Create `zonewarden/src/cli.rs` with `clap` derive struct: `CliArgs { policy, flows, format, output, fail_on_skipped, max_flows }`
3. [ ] Implement `cli::run(args: CliArgs) -> Result<ExitCode, ZonewardenError>`:
   - Reject `max_flows == 0` → `Err(E-SYS-002)`
   - Load policy: `policy::load(&args.policy)` → on error → exit 2
   - Validate: `validator::validate(policy)` → on error → exit 2
   - Open flows file: `File::open(&args.flows)` → on error → exit 2 (E-IO-003)
   - Build `ZeekAdapter::new(file, args.max_flows)`
   - Run pipeline: for each flow from adapter, resolve + classify
   - Aggregate: `aggregator::aggregate(verdicts, policy, skipped, warnings)`
   - Emit warnings to stderr
   - Emit report (format dispatch) → atomic write if `--output`
   - Compute exit code from `ConformanceResult`
4. [ ] Implement `cli::exit_code(result: &ConformanceResult, fail_on_skipped: bool) -> ExitCode`:
   - If `distinct_violating_flows > 0 || idmz_bypasses > 0` → 1
   - If `fail_on_skipped && skipped > 0` → 1
   - Else → 0
5. [ ] Wire `cli::run` in `main.rs`; propagate errors → exit 2
6. [ ] Run `cargo build --release` to produce binary; run integration tests against binary
7. [ ] Verify Red Gate; verify all integration tests pass after implementation
