# S-4.04: Classifier ‚Äî MulticastExempt Short-Circuit + Verdict Totality

## Narrative
- **As a** zonewarden pipeline
- **I want to** short-circuit MulticastExempt flows before any conduit evaluation, and formally verify that every resolved flow receives exactly one VerdictKind
- **So that** multicast traffic does not generate false violations, and the accounting identity (DI-015) is provably maintained

## Acceptance Criteria

### AC-001 (traces to BC-1.04.011 postcondition ‚Äî MulticastExempt short-circuits)
When `dst_kind == DstKind::MulticastBroadcast`, `classifier::classify` returns `VerdictKind::MulticastExempt` immediately, before IntraZone check and before conduit evaluation.
- **Test:** `test_BC_1_04_011_multicast_dst_short_circuits()`

### AC-002 (traces to BC-1.04.011 invariant ‚Äî MulticastExempt is first)
MulticastExempt short-circuits BEFORE IntraZone detection. Even if src and dst zone would be the same, the multicast check takes priority when dst is multicast.
- **Test:** `test_BC_1_04_011_multicast_before_intrazone()`

### AC-003 (traces to BC-1.04.010 postcondition ‚Äî verdict totality)
Every flow that reaches `classifier::classify` receives exactly one `VerdictKind`. The function always returns a `Verdict` value; it never returns `None`, panics, or propagates an error. This is DI-003/DI-015 totality.
- **Test:** `test_BC_1_04_010_verdict_totality_all_inputs()` (Kani harness)

### AC-004 (traces to BC-1.04.011 ‚Äî idmz_bypass false for MulticastExempt)
MulticastExempt flows have `idmz_bypass = false` (DI-016; the single exception to verdict-independence). `idmz::check` is NOT called for MulticastExempt flows.
- **Test:** `test_BC_1_04_011_multicast_exempt_no_idmz_bypass()`

### AC-005 (traces to BC-1.04.010 ‚Äî Verdict type is total)
The `Verdict` struct is not `Option<Verdict>`. `classify()` has return type `Verdict` (not `Option<Verdict>` or `Result<Verdict, _>`). This enforces totality at the type level.
- **Test:** Compile-time verification of function signature

### AC-006 (traces to BC-1.04.011 edge case ‚Äî MulticastExempt not a violation)
`MulticastExempt` flows are NOT counted in `distinct_violating_flows`. They are counted only in `multicast_exempt` tally.
- **Test:** `test_BC_1_04_011_multicast_exempt_not_in_violations()`
